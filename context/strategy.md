# Strategy

## Strategic Principle

Start with **AI usage enablement**, not AI engineering.

Most SMBs are overwhelmed by AI tools but still struggle to apply them in daily work.

The strategy is to solve this gap.

## The Service

AI Work Assistant Setup (90–120 minutes)

This session installs a practical AI workflow for everyday tasks.

## Core Stack

Primary AI brain:
- Claude

Optional comparison tool:
- ChatGPT

Environment:
- VS Code
- Claude Code

Knowledge storage:
- Google Docs or Notion

Local system:
- Mac mini folder structure

No complex automation tools are used initially.

This avoids unnecessary complexity.

## Core Use Cases

The system focuses on 5 universal workflows:

1. Email Assistant
2. Document Summary Assistant
3. Research Assistant
4. Meeting Notes Assistant
5. Brainstorming Partner

These use cases apply to almost all knowledge workers.

## Why This Strategy Works

Reasons:

1. Fast value for clients
2. Low technical barrier
3. Easy to deliver repeatedly
4. Builds trust quickly
5. Opens doors to larger AI projects later

## Delivery Model

Step 1 – Intake questions  
Step 2 – 90–120 min live setup session  
Step 3 – customise prompts for their workflow  
Step 4 – deliver prompt pack and guide

## Key Constraint

Do NOT overbuild the system.

Avoid:

- automation platforms
- agent frameworks
- SaaS development
- complex integrations

Focus only on **AI usage workflows.**

## Success Metric

Success is defined as:

- clients using AI daily
- session easy to deliver
- clients feeling immediate value
- repeatable delivery process
